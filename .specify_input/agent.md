# Agent

This feature implements agent management functionality, allowing users to create, and delete agents.

## Requirements

- All requests to the agent management endpoints SHALL require authentication via a valid JWT token.
  - The JWT token SHALL be provided in the `Authorization` header of the request.
  - The system SHALL validate the JWT token and return a 401 Unauthorized response for requests with invalid or missing tokens.
  - The system SHALL use the `AUTH_SECRET` environment variable to verify the JWT tokens.
  - The decoding of the JWT token SHALL extract the user name (`name`) and email (`email`), which SHALL be used to associate the agent with the authenticated user.

- The system SHALL provide an API endpoint to create a new agent.
  - The path for this endpoint SHALL be `/agents`.
  - The payload for this endpoint SHALL include the following fields:
    - `name`: The name of the agent (string, required).
  - The system SHALL return a 201 Created response with a JSON payload containing the created agent object, including its unique identifier (`id`), name (`name`), and the authenticated user's email (`author`).
  - The system SHALL return a 400 Bad Request response if the `name` field is missing or empty in the request payload.
  - The system SHALL store the agent in the database with the name and the authenticated user's email as the author. ID SHALL be generated by the database.

- The system SHALL provide an API endpoint to retrieve a list of all agents.
  - The path for this endpoint SHALL be `/agents`.
  - The system SHALL return a 200 OK response with a JSON payload containing an array of agent whose `author` field matches the authenticated user's email.
  - The response for this endpoint SHALL include an array of agent objects, each containing the following fields:
    - `id`: The unique identifier of the agent (string).
    - `name`: The name of the agent (string).

## Environment Variables

- `AUTH_SECRET`: A secret key used to verify JWT tokens and authenticate requests.

## Out of Scope

- Integration with the production database (PostgreSQL) is out of scope for this feature. The system SHALL use SQLite for development and testing purposes.
